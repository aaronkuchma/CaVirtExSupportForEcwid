Step 1) After logging in to your ecwid store, click on the "System Settings" tab.

Step 2) Beside the "Credit Card" payment method, click on "[Choose payment processor]".

Step 3) Click the drop down arrow beside "Credit Card", then find "Credit Card: Authorize.NET SIM" in the list and select it.

Step 4) When the Account Details page appears, enter some random characters (letters and numbers) for "API Login Id", "Transaction Key", and "MD5 Hash Value". Keep "Authorize and Capture" as the Transaction Type.

Open up the "config.php" file and change $x_login's value to the API Login Id, and $hash_value's value to the MD5 Hash Value that you selected. Save the file.

Finally, click the "Change" button below "Endpoint URL".

Change the "Endpoint URL" value to point to the domain and path to your "cavirtex_gateway.php" file. Click "Save" when you are done.

Now, click on "Credit card" under the "Payment Method" heading.

Change the first text box's contents to "Bitcoin Payments" and the second text box's contents to "No Credit Card Needed". Click "Apply".

Click the "[ Enable ]" link for "Credit card" in the right most column.

Now, click the checkmark beside "Save" (or press Ctrl-S).

Next, click the "Design" tab, then when you're on the "Design Settings" page click on the "Standard" theme and finally click the "Duplicate" button.

Change the "Theme Name" to "Bitcoins Accepted Here", then add the following custom CSS:

div.ecwid-paymentAuthorizeNet.ecwid-paymentAuthorizeNet {
  background-image: url('https://en.bitcoin.it/w/images/en/7/74/BC_Rnd_64px.png');
  background-repeat:no-repeat;
  margin-left:20px;
  height:64px;
  width:168px;
}

div.ecwid-paymentAuthorizeNet img{
  display: none;
}

You may want to download the bitcoin logo from https://en.bitcoin.it/w/images/en/7/74/BC_Rnd_64px.png and save it to your hosting space, then change the url(' .... '); accordingly.

Click on "Activate" then click the checkbox beside "Save".

That concludes the setup that needs to be performed on the Ecwid site. 

Next is configuring the config.php file.

The $CaVirtEx_MerchantKey and $CaVirtEx_MerchantSecretKey values should be set to whatever is displayed on https://www.cavirtex.com/merchant_information .

$x_login and $hash_value should have been set earlier.

$logDir can be changed to a secure directory on the web server that PHP can write to.

$payment_alert should be changed to your business' e-mail address.

$return_link_url should be changed to point to the "postpayment.php" file on your domain. E.g. 'http://YOURDOMAIN.ca/Ecwid/postpayment.php';

The $proxyXXXX variables should be changed only if you need to use a proxy to access the CaVirtEx website.

Save the file. Upload the files to your domain.

Configuration should now be complete!
